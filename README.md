Graceful degradation is related to service workers in the way that we first need to ensure that the base app over a network works well. Without having this part complete it can be difficult to know what we want to support or not support in other versions. In regard to service workers we can see how important this can be because it can be a matter of just extending our implementation to work well even when we dont have an active network connection. And this is exactly graceful degradation where your app would still be able to run in extended environemnts.
